# WarkeyNETIII-1.1 项目理解总结

## 项目概述

WarkeyNETIII 是一个专为 **魔兽争霸3 (Warcraft III)** 设计的键盘改键工具，是 Warkey.NET II 的升级版，专门针对 Windows 10 进行了优化和重新设计。

## 主要特点

从 README 可以看出项目的核心特性：

1. **不与聊天冲突** - 智能检测聊天状态，避免改键干扰正常聊天
2. **原生 Windows 10 支持** - 需要 Windows 10 周年更新 (build 14393) 或更高版本
3. **现代 UI 设计** - 使用 FluentWPF 将 UWP 控制移植到 WPF，界面美观
4. **高性能** - 最小化时 CPU 占用基本为 0%
5. **无广告** - 纯干净工具
6. **开源** - 完全开放源代码

## 相比 Warkey.NET II 的主要改进

- ✅ 自动以管理员身份运行
- ✅ 完全从头重新编码
- ✅ 采用 UWP 风格重新设计，原生运行在 .NET 4.6.2
- ✅ 使用**键盘钩子 (Keyboard Hook)**替代传统的 GetAsyncKeyState 轮询，性能更好
- ✅ 使用 JSON 序列化和异步方法
- ✅ 添加了自动启动和随魔兽关闭功能
- ✅ 修复了不支持 4K 分辨率的问题
- ✅ 支持多行自动聊天
- ✅ 更高清的物品栏图片

## 核心工作原理

### 1. 键盘钩子原理

使用 Windows 原生 API `SetWindowsHookEx` 设置**低级键盘钩子 (WH_KEYBOARD_LL)**，全局监听键盘事件：
- 当按键按下 (`WM_KEYDOWN`) 触发普通热键
- 当系统按键按下 (`WM_SYSKEYDOWN`) 且不是 Alt 本身时，触发 Alt+ 组合热键
- 只有当魔兽争霸3在前景时才处理按键

**核心代码逻辑（KeyboardHookService）：**
```csharp
if (MainService.IsWar3Foreground && !ChatboxService.IsChatting && nCode >= 0) {
    // 解析按键并触发 GlobalKeyDown 事件
}
```

### 2. 改键工作流程

1. **检测魔兽窗口**: `MainService` 每秒检查一次魔兽窗口是否存在
   - 如果不存在且启用了"随魔兽关闭"，自动关闭程序
   - 如果存在，记录窗口句柄并标记状态

2. **全局按键监听**: `KeyboardHookService` 钩子捕获按键事件

3. **按键匹配处理** (`MainService.KeyboardHookService_GlobalKeyDown`):
   - **物品栏改键**: 检查是否匹配用户设置的 6 个物品栏热键，如果匹配则通过 `PostMessageService` 向魔兽发送对应小键盘按键消息
   - **自动聊天**: 检查是否匹配自动聊天热键，如果匹配则通过 `ChatboxService` 发送预设消息
   - **快速载入**: 如果按下 F8，自动发送 `-load` 命令和保存密码

### 3. 改键实现细节

`PostMessageService.PostItemMessageToWar3`:
- 默认映射到小键盘按键：`7, 8, 4, 5, 1, 2` (对应 6 个物品栏位置)
- 使用 Windows `PostMessage` API 直接向魔兽窗口发送 `WM_KEYDOWN` 和 `WM_KEYUP` 消息
- 如果用户使用的是 Alt+ 组合，需要特殊处理 Alt 按键的状态

### 4. 自动聊天实现

`ChatboxService.SendMessageToChatbox`:
- 使用 `SendKeys.SendWait()` 模拟按键输入：`{ENTER} + 消息 + {ENTER}`
- 设置 `IsChatting` 标志防止改键在发送聊天消息时触发，**避免冲突**
- 发送完成后重置标志

## 主要功能模块

### 1. 物品栏改键 (Warkey 功能)
- 支持 6 个自定义热键（对应 6 个物品栏位置）
- 支持普通按键和 Alt+ 组合键
- 将自定义按键转换为魔兽识别的小键盘按键

### 2. 自动聊天 (AutoChat)
- 用户可以预设多条聊天消息
- 每条消息绑定一个热键
- 按下热键自动发送消息到聊天框
- 支持多行消息

### 3. 自动载入存档 (LoadGame)
- 保存游戏存档密码
- 按下 F8 自动输入 `-load` 命令和密码
- 方便 Dota 等地图快速载入存档

### 4. 设置功能
- 开机最小化
- 自动启动魔兽（如果war3.exe在同目录）
- 随魔兽关闭自动退出
- 所有设置保存为 JSON 文件

## 技术栈

- **平台**: .NET Framework 4.6.2 (WPF)
- **UI 框架**: FluentWPF（现代流畅设计）
- **序列化**: Newtonsoft.Json
- **编译优化**: Costura.Fody（嵌入依赖 DLL 到 exe）
- **Windows API**: 使用 P/Invoke 调用 user32.dll 原生方法

## 技术亮点

1. **低CPU占用**: 使用键盘钩子而非轮询，空闲时 CPU 占用接近 0
2. **避免聊天冲突**: 发送消息时设置标志，改键不会在聊天时触发
3. **异步IO**: 异步读写设置文件，不卡 UI
4. **自动资源管理**: 退出时正确卸载键盘钩子，避免内存泄漏
5. **现代化设计**: 使用 MVVM 架构，代码结构清晰
6. **Json 配置**: 配置文件易于读写和备份

## 核心工作原理

### 1. 键盘钩子原理

使用 Windows 原生 API `SetWindowsHookEx` 设置**低级键盘钩子 (WH_KEYBOARD_LL)**，全局监听键盘事件：
- 当按键按下 (`WM_KEYDOWN`) 触发普通热键
- 当系统按键按下 (`WM_SYSKEYDOWN`) 且不是 Alt 本身时，触发 Alt+ 组合热键
- 只有当魔兽争霸3在前景时才处理按键

**核心代码逻辑（KeyboardHookService）：**
```csharp
if (MainService.IsWar3Foreground && !ChatboxService.IsChatting && nCode >= 0) {
    // 解析按键并触发 GlobalKeyDown 事件
}
```

### 2. 改键工作流程

1. **检测魔兽窗口**: `MainService` 每秒检查一次魔兽窗口是否存在
   - 如果不存在且启用了"随魔兽关闭"，自动关闭程序
   - 如果存在，记录窗口句柄并标记状态

2. **全局按键监听**: `KeyboardHookService` 钩子捕获按键事件

3. **按键匹配处理** (`MainService.KeyboardHookService_GlobalKeyDown`):
   - **物品栏改键**: 检查是否匹配用户设置的 6 个物品栏热键，如果匹配则通过 `PostMessageService` 向魔兽发送对应小键盘按键消息
   - **自动聊天**: 检查是否匹配自动聊天热键，如果匹配则通过 `ChatboxService` 发送预设消息
   - **快速载入**: 如果按下 F8，自动发送 `-load` 命令和保存密码

### 3. 改键实现细节

`PostMessageService.PostItemMessageToWar3`:
- 默认映射到小键盘按键：`7, 8, 4, 5, 1, 2` (对应 6 个物品栏位置)
- 使用 Windows `PostMessage` API 直接向魔兽窗口发送 `WM_KEYDOWN` 和 `WM_KEYUP` 消息
- 如果用户使用的是 Alt+ 组合，需要特殊处理 Alt 按键的状态

### 4. 自动聊天实现

`ChatboxService.SendMessageToChatbox`:
- 使用 `SendKeys.SendWait()` 模拟按键输入：`{ENTER} + 消息 + {ENTER}`
- 设置 `IsChatting` 标志防止改键在发送聊天消息时触发，**避免冲突**
- 发送完成后重置标志
