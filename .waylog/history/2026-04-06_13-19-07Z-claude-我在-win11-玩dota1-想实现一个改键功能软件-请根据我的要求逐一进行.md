## WarkeyNETIII-1.1 项目总结

### 用途
为 **Dota 1（Warcraft III）** 玩家提供改键功能，将自定义按键映射到游戏 6 个物品槽，并支持自动发送聊天消息。

---

### 技术栈
| 项目 | 说明 |
|------|------|
| .NET Framework 4.6.1 | 目标运行时 |
| WPF | UI 框架 |
| MVVM | 架构模式 |
| FluentWPF 0.7 | UWP 亚克力风格 UI |
| Newtonsoft.Json | 设置序列化 |
| Costura.Fody | 打包成单个 exe |

---

### 核心机制

**键盘拦截**
- `KeyboardHookService`：通过 `SetWindowsHookEx(WH_KEYBOARD_LL)` 安装全局低级键盘钩子
- 仅在 War3 窗口处于前台且不在聊天时生效

**按键转发**
- `PostMessageService`：用 `SendMessageA` 向 War3 窗口句柄发送 `WM_KEYDOWN/KEYUP`
- 6 个物品槽固定映射到 **NumPad 7/8/4/5/1/2**

**War3 进程检测**
- `MainService` 每秒轮询 `war3`/`War3` 进程，获取窗口句柄
- `ForegroundWindowService`：`GetForegroundWindow()` 判断是否在前台

---

### 功能模块

| 模块 | 功能 |
|------|------|
| **Warkey** | 6 个物品槽自定义改键，支持 `Alt+键` |
| **AutoChat** | 绑定快捷键一键发送聊天文字（多行支持） |
| **LoadGame** | F8 自动发送 `-load` 存档密码（针对 TKOK 地图） |
| **Settings** | 开机最小化、随 War3 启动/关闭、分辨率优化、锁帧设置 |

---

### 架构特点
- `MainWindow` 持有所有 ViewModel 的静态引用，供 `Services` 层直接访问（非依赖注入）
- 设置以 JSON 文件 `WarkeyNETIII.json` 存储在 exe 同目录
- `app.manifest` 要求管理员权限（键盘钩子需要）
- 聊天输入检测通过 `ChatboxService.IsChatting` 标志位防止改键与打字冲突

### 为什么原版只有 1.28MB

| 项目 | 原因 |
|------|------|
| **.NET Framework 4.6.1** | Windows 10/11 **系统自带**，exe 里完全不需要打包运行时 |
| **Costura.Fody** | 把 Newtonsoft.Json、FluentWPF 等依赖压缩嵌入 exe，几乎无损耗 |
| 最终 exe | 只含应用代码 + 压缩依赖 ≈ 1.28MB |

### 为什么我们是 165MB

```
--self-contained true
```
这个参数把整个 **.NET 10 运行时**（BCL + WPF + WinForms + JIT）全部打包进去了，因为 Win11 默认没有安装 .NET 10。

---

### 解决方案对比

| 方案 | exe 大小 | 前提条件 |
|------|----------|----------|
| 当前（self-contained） | ~165MB | 无需任何安装 |
| **framework-dependent** | **~400KB** | 目标机器需安装 .NET 10 Desktop Runtime |
| self-contained + 裁剪 | ~30-50MB（WPF 裁剪效果有限） | 无需安装，但可能裁掉反射代码 |

我们的 `Settings.cs` 用了反射（`GetProperty`），裁剪会出问题，所以最稳的方案是 **framework-dependent**。

### 为什么原版是 1.28MB

`.NET Framework 4.6.1` 是 **Windows 10/11 系统内置**的，exe 里只需要放应用代码。就像写一个调用系统 API 的程序，不需要把整个操作系统打包进去。Costura.Fody 再把少量 NuGet 依赖压缩嵌入，最终只有 1.28MB。

---

### .NET 10 的三种选择

| 方案 | exe 大小 | 条件 |
|------|----------|------|
| self-contained 单文件（当前） | **165MB** | Win11 无需任何安装，双击即用 |
| framework-dependent 单 exe | **159KB** | Win11 需先安装 .NET 10 Desktop Runtime（免费，约 50MB） |
| framework-dependent 多文件 | 159KB + 若干 DLL | 同上，需整个文件夹一起复制 |

---

### 建议

**你的场景（自用）推荐 framework-dependent 159KB 版**：
1. 在 Win11 上安装一次 [.NET 10 Desktop Runtime](https://dotnet.microsoft.com/download/dotnet/10.0)（只装一次，以后所有 .NET 10 程序都能用）
2. 用这个命令生成的 `publish-fd/Dota1Warkey.exe`（159KB）即可

如果要**分发给别人**用，用 165MB 的 self-contained 版，对方不需要安装任何东西。

**没有办法像 .NET Framework 那样做到 1MB 级别**，因为 .NET 10 不内置于 Windows，这是框架设计的根本差异。

因为 **.NET Framework 4.6.1 已经内置在 Windows 10/11 里**，不是需要额外安装的东西。

---

### 时间线

| 时间 | 事件 |
|------|------|
| 2015 年 | .NET Framework 4.6 随 **Windows 10** 一起发布，作为系统组件内置 |
| 2016 年 | .NET Framework 4.6.1 通过 Windows Update 推送到所有 Win10 |
| 现在 | Win10 / Win11 **出厂自带** .NET Framework 4.6.1，任何人的电脑上都有 |

---

### 而 .NET 10 不同

.NET 10（也叫 .NET Core 的后续版本）是微软 2019 年重新设计的**跨平台新框架**，设计目标是 Linux / macOS / Windows 都能跑，所以**不内置于任何操作系统**，需要单独安装。

---

### 所以本质区别是

```
.NET Framework 4.6.1  →  Windows 的一部分，像 DirectX 一样内置
.NET 10               →  独立软件，需要单独安装（或打包进 exe）
```

原作者选 .NET Framework 4.6.1 正是出于这个考虑——用户双击就能跑，无需任何前置条件。

---

你现在有两条路：
1. **继续用 .NET 10**，发给别人时用 165MB 自包含版，或要求对方装一次 Runtime
2. **改回 .NET Framework 4.8**（Win11 内置，比 4.6.1 更新），可以做到和原版一样小，且无需任何安装

根本原因：我们在匹配改键时会**抑制原始按键**（返回 `(IntPtr)1`），导致聊天时输入的字符也被吃掉。

原版靠 `ChatboxService.IsChatting` 标志位解决，但那只处理自动聊天。真正的解法是**追踪 Enter/Esc 来感知聊天框开关状态**，聊天中完全不拦截。

**Claude**

完成，541KB 单文件。

**逻辑说明：**

| 按键 | 行为 |
|------|------|
| Enter | 切换 `_isChatting`（`false→true` 打开，`true→false` 发送）|
| Esc | 强制 `_isChatting = false`（取消聊天）|
| 聊天中任意键 | 直接透传，完全不拦截 |
| 非聊天时改键 | 拦截并发 NumPad，抑制原始键 |
